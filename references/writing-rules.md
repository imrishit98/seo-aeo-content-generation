# Writing Rules

Rules for producing content that reads as human-written, matches the site's regional English dialect, and passes AI detection scrutiny. Apply these rules to every piece of content without exception.

---

## Em Dash Ban

**Absolute rule: No em dashes in any output. Zero tolerance.**

The em dash (--) is the single most reliable marker of AI-generated content. AI models overuse em dashes because their training data (books, academic papers, style guides) contains them frequently. Most human writers rarely use em dashes because the character does not appear on a standard keyboard.

### Replacements

| Instead of | Use |
|------------|-----|
| The results -- which were surprising -- showed... | The results, which were surprising, showed... |
| This approach -- unlike traditional methods -- works | This approach (unlike traditional methods) works |
| Three factors matter -- speed, cost, and quality | Three factors matter: speed, cost, and quality |
| The tool is powerful -- and free | The tool is powerful, and it costs nothing |
| Data shows -- as expected -- that usage grew | Data shows, as expected, that usage grew |

### Self-Check

After generating content, search for `--` and `—` in the output. If either appears, replace it. No exceptions, no "just this once."

## AI-Tell Word Blacklist

These words and phrases trigger AI detection tools and make content sound artificial. Never use them.

### Banned Verbs

| Banned | Alternatives |
|--------|-------------|
| delve (into) | explore, examine, investigate, look at, dig into |
| leverage | use, apply, draw on, take advantage of |
| utilize | use |
| facilitate | help, enable, support, make possible |
| foster | encourage, support, develop, grow, nurture |
| bolster | strengthen, support, reinforce, back up |
| underscore | highlight, stress, show, point out |
| unveil | reveal, show, introduce, release, announce |
| navigate | manage, handle, work through, deal with |
| streamline | simplify, speed up, make more efficient, cut steps from |
| enhance | improve, strengthen, boost, raise |
| endeavour | try, attempt, work toward |
| ascertain | find out, determine, confirm, check |
| elucidate | explain, clarify, spell out, break down |
| optimize/optimise | improve, refine, tune, sharpen (use optimize/optimise only when discussing SEO or technical optimization specifically) |

### Banned Adjectives

| Banned | Alternatives |
|--------|-------------|
| robust | strong, reliable, solid, thorough, dependable |
| comprehensive | complete, thorough, full, detailed, end-to-end |
| pivotal | key, critical, central, important, deciding |
| crucial | important, key, essential, necessary |
| vital | important, essential, necessary |
| transformative | significant, major, game-changing |
| cutting-edge | new, advanced, recent, modern, latest |
| groundbreaking | new, original, significant, first-of-its-kind |
| innovative | new, original, creative, fresh |
| seamless | smooth, easy, effortless, frictionless |
| intricate | complex, detailed, complicated |
| nuanced | subtle, complex, detailed, layered |
| multifaceted | complex, varied, diverse |
| holistic | complete, whole, full-picture, all-in-one |

### Banned Transitions and Connectors

| Banned | Alternatives |
|--------|-------------|
| furthermore | also, and, plus, on top of that |
| moreover | also, and, besides, what's more |
| notwithstanding | despite, even so, still, regardless |
| that being said | however, but, still, then again |
| at its core | essentially, fundamentally, really, at the heart of it |
| to put it simply | in short, simply put, put another way |
| it is worth noting that | note that, importantly, keep in mind |
| in the realm of | in, within, when it comes to |
| in the landscape of | in, within, across |
| in today's [anything] | currently, now, today, right now |

### Banned Opening Phrases

Never start content with any of these:
- "In today's fast-paced world..."
- "In today's digital age..."
- "In an era of..."
- "In the ever-evolving landscape of..."
- "In the realm of..."
- "It's important to note that..."
- "Let's delve into..."
- "Imagine a world where..."
- "Picture this..."
- "Have you ever wondered..."

### Banned Transitional Phrases

Never use mid-content:
- "That being said..."
- "With that in mind..."
- "It's worth mentioning that..."
- "At its core..."
- "To put it simply..."
- "In essence..."
- "This begs the question..."
- "Let's explore..."
- "Now, let's dive into..."

### Banned Closing Phrases

Never end content with:
- "In conclusion..."
- "To sum up..."
- "By [doing X], you can [achieve Y]..."
- "In the final analysis..."
- "All things considered..."
- "At the end of the day..."
- "The bottom line is..."

### Banned Structural Patterns

- "Whether you're a [X], [Y], or [Z]..." (listing three examples after "whether")
- "It's not just [X], it's also [Y]..."
- "Think of [X] as [elaborate metaphor]..."
- Starting sentences with "By" followed by a gerund: "By understanding X, you can Y..."

### Banned Filler Words

Remove these unless they add specific meaning:
- absolutely, actually, basically, certainly, clearly, definitely
- essentially, extremely, fundamentally, incredibly, interestingly
- naturally, obviously, quite, really, significantly, simply
- surely, truly, ultimately, undoubtedly, very

### Banned Academic AI-Tells

| Banned | Alternatives |
|--------|-------------|
| shed light on | clarify, explain, reveal, show |
| pave the way for | enable, allow, make possible, open the door to |
| a myriad of | many, numerous, various, dozens of |
| a plethora of | many, numerous, several, plenty of |
| paramount | very important, essential, critical, top-priority |
| pertaining to | about, regarding, on, related to |
| prior to | before |
| subsequent to | after |
| in light of | because of, given, considering, due to |
| with respect to | about, regarding, for, on |
| in terms of | regarding, for, about, when it comes to |
| the fact that | that (or rewrite the sentence entirely) |

## Sentence and Paragraph Rules

### Paragraphs

- Maximum 2-3 sentences per paragraph
- No single-sentence paragraphs unless used for deliberate emphasis (limit to 1-2 per piece)
- No 4+ sentence paragraphs ever

### Sentence Variety

- Vary sentence length deliberately. Mix short (5-10 words) with medium (15-20 words) and occasional longer (25-30 words)
- Never place more than 2 consecutive sentences of similar word count
- Alternate between sentence structures: subject-verb, dependent clause opener, question
- Read the content aloud mentally. If it has a rhythmic, predictable cadence, vary the structure

### Tone Test

Ask these questions about every paragraph:

1. Would a knowledgeable professional say this in a conversation with a peer?
2. Does this sound like something a real person would write in a company blog?
3. If read aloud, would listeners notice it sounds "off" or robotic?

If the answer to question 3 is yes, rewrite the paragraph.

## Regional English Matching

All content must match the regional English dialect of the target site. This includes spelling, vocabulary, idioms, date formats, and measurement units.

### Detection

When you have access to existing site content:

1. **Check HTML:** Look for `<html lang="en-US">`, `<html lang="en-GB">`, `<html lang="en-AU">`, etc.
2. **Spelling scan:** Check for -ise vs -ize, -our vs -or, -re vs -er endings
3. **Vocabulary scan:** Look for regional terms (see table below)
4. **Date format:** MM/DD/YYYY vs DD/MM/YYYY vs YYYY-MM-DD

| Dialect | Currency Format | Measurement |
|---------|----------------|-------------|

| Australian | A$X.XX AUD | Metric (metres, kilograms, Celsius) |
| Canadian | C$X.XX CAD | Metric (metres, kilograms, Celsius) |

### Application Rules

1. Once a dialect is identified, apply it with 100% consistency. No mixing.
2. If dialect cannot be determined from site content, default to **American English** and note this in the output.
3. When writing about a topic that has no regional variation (e.g., technical API documentation), maintain the site's dialect in surrounding text while keeping technical terms standard.
4. Company names, product names, and proper nouns are never modified for dialect.

## Tone Matching

### How to Match Existing Site Voice

When site content is available, analyze:

1. **Formality level:** Does the site use "you" and "we" (conversational) or third-person language (formal)?
2. **Jargon usage:** Does the site assume technical knowledge or explain everything?
3. **Sentence style:** Short and punchy? Or longer and explanatory?
4. **Humor:** Does the site use humor or stay strictly professional?
5. **First vs third person:** Does the site say "we help you" or "[Company] helps businesses"?

Match whatever pattern the existing content uses. If no existing content is available, default to:
- Second person ("you")
- Professional but approachable
- Industry-appropriate jargon (explain on first use)
- No humor unless the user requests it

## Post-Writing Self-Check

Run through these checks before delivering content:

1. Search for `--` and `—` (em dashes). Replace all.
2. Search for every word in the banned verbs, adjectives, and phrases lists. Replace all.
3. Count sentences per paragraph. Fix any with 4+ sentences.
4. Check for 3+ consecutive sentences of similar length. Vary them.
5. Verify regional English consistency. Check 10 random words with regional variants.
6. Read the opening paragraph. Does it sound like a human expert or a language model?
7. Read the closing paragraph. Does it end with a banned closing phrase?
8. Check for filler words. Remove any that add nothing to the meaning.
